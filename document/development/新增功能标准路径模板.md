# 新增功能标准路径模板

本模板用于新增或调整一个前后端功能时固定实施顺序，减少遗漏装配、契约生成和测试补位。

## 一、后端路径

1. Controller
   - 新增或调整 `backend/src/modules/*/controllers/*` 路由。
   - 所有 `/api/v1/*` 路由必须声明 `response_model`。
   - 返回统一响应外壳，不在 Controller 内拼事务。

2. Service
   - 业务编排进入 `backend/src/modules/*/services/*`。
   - 写路径必须明确事务边界、版本推进、outbox 事件和错误码。
   - Controller 只做鉴权上下文、参数模型和 service 调用。

3. Repository
   - 接口定义放在 `backend/src/repositories/base` 或 `backend/src/repositories/query`。
   - SQLAlchemy 实现放在 `backend/src/infrastructure/db/repositories`。
   - 模块层不得直接导入数据库连接、SQLAlchemy `text` 或 repository support 私有工具。

4. Container / Module 注册
   - 新 repository getter 放入对应 `backend/src/app/containers/*_repositories_container.py`。
   - 新 service getter 放入对应领域 container。
   - 新 router 统一通过 `backend/src/app/modules.py` 的 `APP_MODULES` 注册。
   - 有后台任务时必须同时声明 startup/shutdown hook。

## 二、前端路径

1. API 类型与调用层
   - 后端 schema 变化后运行 `npm run generate:api-types`。
   - 前端 API 调用只放在 `frontend/src/api/*` 或领域 hook 中。
   - 页面组件不得直接调用 `fetch`、硬编码 `/api/*`、或直接导入 API client。

2. View-model / Model
   - DTO 到 UI 展示模型转换放入 `frontend/src/view-models/*` 或领域 model。
   - 页面组件只消费已适配的 view-model、controller hook 返回值或组件 props。
   - 复杂派生逻辑必须写成纯函数并补单测。

3. Page / Component
   - 页面负责路由、布局和组合。
   - 业务流程进入 `use*Controller` / 领域 hook。
   - 重复卡片、表单、面板逻辑进入组件，不在页面内堆分支。

## 三、契约与门禁

1. 本地快速准入：`make check`。
2. 全栈烟测准入：`make smoke`。
3. `make check` 会执行契约生成并检查以下生成物无 diff：
   - `backend/openapi.json`
   - `backend/realtime.schema.json`
   - `frontend/src/api/types.generated.ts`
   - `frontend/src/ws/realtime.generated.ts`

## 四、测试补位

1. 后端
   - Service 写路径补单元测试。
   - API envelope、错误码、关键路径补集成测试。
   - 新模块边界不得破坏 `test_architecture_boundaries.py`。

2. 前端
   - API adapter / view-model / 纯 model 补单元测试。
   - Hook 流程补 `renderHook` 测试。
   - 页面层不得绕过 `frontendBoundaries.test.ts` 约束。

3. E2E
   - 影响首页、设置保存、设备控制、编辑发布、WS 重连时，必须补或更新 Playwright 用例。
   - E2E 不默认放入 `make check`，统一通过 `make smoke` 执行。

## 五、完成标准

1. `make check` 全绿。
2. 涉及主链路时 `make smoke` 全绿或记录无法运行原因。
3. 生成物无漂移。
4. 新增功能有明确 Controller、Service、Repository、前端 API、view-model 和测试落点。
